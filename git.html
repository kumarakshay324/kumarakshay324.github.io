Version Control using Git


Version control is the most important system for managing code bases. Version control is the system of recording different versions of files, usually source code files, changing over time. It allows users to track changes in code over time and helps collaborative code development. Tracking and unique identification of every different change posted to a code enables users to completely or selectively revert to older versions when and if necessary.

Version control system (VCS) is a very handy software for code organization, multiple developer collaboration and code sharing. VCS allows parallel code development/testing/deployment for several programmers spread across multiple large teams.

Git is one of the best and most widely used distributed version control system to handle projects. It was developed by Linus Torvalds during the development of the Linux kernel. It is a free and open-source tool shipped natively with most of the linux distros. While local installtion of git is sufficient for individual purpose, the true essence of VCS lies in remote collaboration.



INTRODUCTION

Git is a decentralized version control system where all users can have their individual versions of the code. Also, git allows offline code development while network connectivity and online VCS hosts are needed for collaboration. Online platforms like Github, Gitlab and Bitbucket are the most widely used ones. Git keeps variations in code over time by recording and tracking only the difference in code and not maintaining multiple code copies.

GIT CONCEPTS

There are few basic concepts of git to let user access, modify, develop and propose changes to the project source code.

  • Repository - Project source code folder
  • Branch - Different version of the same source code that can different users can use, modify and makes changes to, independently
  • Fork/Clone - Create a personal copy of the project source code, essentially from an online hosting platform
  • Commit - Add the new changes to the source code, to the current version aka branch and log it with a message and a timestamp
  • Remote - Online repository storage

GENERAL GIT COMMANDS

General Commands


⇒ Initialize the git control system inside the project source code folder
  git init .

⇒ Stage the new addtional changes completely or partially to commit to the branch
  git add . // Stage all changes
  git add --all // Stage changes including deleted files
  git add <individual files or folder> // Stage all changes

⇒ Finally add staged changes to the current branch
  git commit -m <message to notify commit log>

⇒ Push the new commits on a local branch to remote repository
  git push origin <branch> //

⇒ Clone a remote repository to a local location
  git clone remote_git_url

⇒ Check the difference and edits in new uncommitted files

Branch Commands


⇒ Switch to the specified branch
  git checkout <branch_name> //


⇒ Create a new branch
  git checkout -b <new_branch_name> <base_branch_name>

⇒ Delete a branch
  git branch -d <branch_name>

⇒ Switch to a particular commit ID of the code if it exists on the branch
  git checkout <commit_id>

Discard or archive edits


⇒ Remove all new unwanted edits
  git stash


⇒ Obtain the list of stashed or archived changes
  git stash list

⇒ Archive the current uncommitted changes to stash
  git stash save "<message_to_save_edit>"

⇒ Apply last stashed set of changes to edits
  git stash apply

⇒ Apply a particular set of stashed changes to the current branch
  git stash apply stash@{<stash>} // After looking at the stash number from the list

Configure user credentials


⇒ Add username for remote git service account
  git config --global user.name "<github/bitbucket user_name>"


⇒ Add email ID for remote git service account
  git config --global user.email "<github/bitbucket user_mail_id>"
