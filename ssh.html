Basics of SSH Usage


SSH or secured shell is a network protocol to communicate between multiple machines securely. Any system administration service, file transfer and network applications can be performed over SSH. The access through a command-line interface to another machine's shell (I use bash on Ubuntu) is the primary mode of interaction. It is a secure channel that commnunicates using a client-server architecture. The default TCP port used is 22 but others can also be used.

This article is a location to aggregate all the methods commonly used for shell access, SSH-key deployment for repositories and account, SSH-tunneling, file transfer over SSH and using SSH in VS Code IDE for software development on other machines on the local network. All discussions in this article are made for SSH protocol version 2.

The .ssh. folder in Linux machines stores all the important configurations and security keys. The most common files configuration files are config, authorized_keys and known_hosts.

  • The config file has configuration for different services like git that use the SSH credentials for various connections.
  • The known_hosts file has a list of public SSH keys of all the machines currently or previously connected to.
  • The authorized_keys file has the public SSH keys of the machines that it shall accept incoming SSH connections from. This is a highly safe access control feature that ensures no unwarranted client machine can access the server machine.

SSH Command Line Access

SSH command line access is the simplest methods for machines to access the server machine. Common SSH client software include Putty (Windows & Linux), OpenSSH (I used this on Ubuntu), WinSCP (Windows) and CyberDuck (Mac) among others. The basic commands for command-line access are:

  • ssh <machine_username>@<machine_ip_address>
    Access the other machine's shell directly using port 22
  • ssh <machine_username>@<machine_ip_address> -p <port_number>
    Access the other machine's shell directly using any port number other than 22.
  • ssh <machine_username>@<machine_ip_address> -i <identity_file>
    Access the other machine's shell directly using a specific identity file, a file with a private key. By default, a local network connection uses the .ssh/id_rsa private key of the machine (on Linux machines) for connections. SSH connection to online services generally uses a .pem file as an identity file.

Here, the N nodes are laid out along the rows and the columns of a N x N matrix adjacency_matrix. Now, for a pair of nodes that have an undirected edge, the value of adjacency_matrix(nodeA, nodeB) and adjacency_matrix(nodeB, nodeA)is non-zero and often used to represent the weight of the edge. For a directed graph, from node A to node B, only adjacency_matrix(nodeA, nodeB) value is non zero and the other pair adjacency_matrix(nodeB, nodeA) is zero (rows and column node represents the start and end node respectively).
